# 提示词优化程序需求文档

## 摘要

- 项目目标：提供一个单人使用的提示词评估与优化工具。
- 核心能力：评估提示词、优化提示词、管理模型配置、保存优化结果。
- 设计原则：流程简单、配置可控、结果可追溯。

## 范围与目标

- In Scope：提示词评估、提示词优化、模型与服务商管理、优化记录保存与查看。
- Out of Scope：多用户协作、权限系统、复杂工作流编排。

## 用户操作流程

1. 用户输入提示词，点击“评估”。
2. 前端请求后端评估接口，后端返回评估结果。
3. 用户点击“优化”。
4. 前端请求后端优化接口，后端返回优化后的提示词。
5. 用户可复制评估结果和优化结果。
6. 系统保存本次优化前后提示词及相关配置。

## 功能需求

### 1) 模型设置页面

| 编号 | 需求 |
| --- | --- |
| MS-01 | 提供独立的“模型设置”页面。 |
| MS-02 | 用户可启用/禁用模型服务商。 |
| MS-03 | 用户可启用/禁用服务商下的模型。 |
| MS-04 | 服务商可配置 `baseUrl` 与 `apiKey`。 |
| MS-05 | 支持手动拉取模型列表，并将结果缓存到数据库。 |
| MS-06 | 支持手动添加模型到数据库。 |
| MS-07 | 可分别设置“评估默认模型”与“优化默认模型”。 |

### 2) 提示词优化页面

| 编号 | 需求 |
| --- | --- |
| PO-01 | 可分别为“评估”和“优化”步骤指定模型。 |
| PO-02 | 可分别配置模型参数（如 `temperature` 等）。 |
| PO-03 | 支持复制评估结果。 |
| PO-04 | 支持复制优化后的提示词。 |
| PO-05 | 优化完成后保存优化前后提示词。 |

### 3) 优化结果保存页面

| 编号 | 需求 |
| --- | --- |
| SP-01 | 提供“优化过的提示词保存页面”。 |
| SP-02 | 页面仅展示优化后的提示词列表。 |
| SP-03 | 每条记录可追溯到来源优化任务（用于审计/排查）。 |

## 数据与存储要求

| 对象 | 关键字段 |
| --- | --- |
| 服务商 Provider | `id` `name` `enabled` `baseUrl` `apiKey` `createdAt` `updatedAt` |
| 模型 Model | `id` `providerId` `modelName` `enabled` `createdAt` `updatedAt` |
| 默认模型配置 DefaultModelConfig | `evaluateModelId` `optimizeModelId` |
| 优化任务 PromptRun | `id` `originalPrompt` `evaluationResult` `optimizedPrompt` `evaluateModelId` `optimizeModelId` `evaluateParams` `optimizeParams` `createdAt` |
| 优化结果 SavedPrompt | `id` `promptRunId` `optimizedPrompt` `createdAt` |

## 公共接口/类型变更（建议约束）

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/prompt/evaluate` | `POST` | 评估提示词，返回评估结果。 |
| `/api/prompt/optimize` | `POST` | 优化提示词，返回优化文本。 |
| `/api/providers` | `GET/PUT` | 查询与更新服务商配置及启用状态。 |
| `/api/providers/{id}/models/sync` | `POST` | 手动拉取并缓存模型列表。 |
| `/api/models` | `POST/PUT` | 手动新增模型、更新模型启用状态。 |
| `/api/model-defaults` | `GET/PUT` | 读取/设置评估与优化默认模型。 |
| `/api/saved-prompts` | `GET` | 查询已保存优化提示词。 |

## 验收测试场景

1. 输入提示词后可成功完成“评估 -> 优化”完整链路。
2. 未显式选择模型时，系统正确使用默认评估模型与默认优化模型。
3. 显式指定模型时，系统按用户指定模型执行，不被默认值覆盖。
4. 服务商禁用后，其模型不可用于评估和优化。
5. 手动拉取模型后，模型列表写入数据库并可在页面展示。
6. 每次优化任务均保存优化前后提示词。
7. 保存页面仅展示优化后的提示词，不展示原始提示词正文。
8. 评估结果与优化结果均可一键复制。

## 假设与默认值

- 单人使用，无登录与权限分级。
- `apiKey` 需要安全存储（至少加密或脱敏展示）。
- 默认支持基础模型参数：`temperature`、`maxTokens`。
- “保存页面只展示优化后提示词”与“系统保存优化前后文本”同时成立：前者是展示策略，后者是存储策略。
